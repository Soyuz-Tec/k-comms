# Secure Development Lifecycle

- Threat model material changes before implementation.
- Security acceptance criteria accompany sensitive stories.
- Static analysis, dependency scanning, secret scanning, image scanning, and IaC scanning run in CI.
- High-risk changes require security review.
- Penetration testing precedes production launch and major trust-boundary changes.
- Vulnerabilities have severity-based remediation objectives and tracked exceptions.
- Security controls are validated in staging and monitored in production.
