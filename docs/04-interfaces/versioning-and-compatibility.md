# Versioning and Compatibility

- Major API versions are explicit in paths or negotiated protocols.
- Event names include a major schema version.
- Servers support a documented minimum client version.
- A compatibility window covers at least one safe rollback release.
- Database changes follow expand/contract sequencing.
- Provider integrations have contract tests and sandbox verification.
- Deprecations include announcement, telemetry, migration guidance, and removal date.
